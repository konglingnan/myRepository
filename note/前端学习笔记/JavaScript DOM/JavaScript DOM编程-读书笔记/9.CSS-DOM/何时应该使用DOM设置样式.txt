







	正常情况下我们不应该用DOM设置样式，但css不方便的时候我们还是可以yongDOM设置样式的。
		
		那么何时应该使用DOM设置样式：


		1.需要根据元素在节点树里的位置设置。

		2.根据某种条件反复设置某种样式。

		3.响应事件（css难以部署时，比如不是所有浏览器都支持所有元素的:hover，在这样的场合DOM却能公平对待。绝大多数的现在浏览器虽然对css伪类的支持很不完整，但对DOM却都有着良好的支持。在浏览器们对css的支持进一步完善之前，在事件发生时用DOM改变html元素的样式更切合实际。）



		这几种情况都属于javascript入侵css领地的情况，我们这么做的理由是：
			1.css无法让我们找到想要处理的目标元素。

			2.css寻找目标元素的办法还未得到广泛的支持。
