






	在创建基于浮动的布局时，一般将两列都向左浮动，然后使用外边距或者内边距在两列之间创建一个隔离带。在使用这种方法时，列在可用空间内包的很紧，没有喘息的空间。如果浏览器表现良好的话，没有问题。但有一些浏览器（比如说IE）会打乱紧密的布局，迫使一列退到另一列的下面。


	因为IE考虑元素内容的尺寸，而不是元素本身的尺寸。在符合标准的浏览器中，如果元素内容太大，它只会超出框外。而IE会使整个元素扩展（可能只是由于将文本设置为斜体）。这样就会打乱紧密的布局。。


	为了防止发生这种情况。我们可以不适用水平外边距来建立隔离带，而是让一个元素左浮动，另一个元素右浮动，从而建立视觉上的隔离带。如果一个元素意外增加了几个px，那么它不会立刻占满水平空间并下降，而只是扩展到视觉隔离带中。


	e.g.

		<div class="content">
			<div class="primary"></div>
			<div class="secondary"></div>
		</div>


		.content {
			width: 1000px;
		}

		.content .primary {
			float: right;
			width: 590px;
		}
		.content .secondary {
			float: left;
			width: 390px;
		}
